---
title: "Agent templates"
description: "Define reusable configurations for launching agents"
---

Agent templates define the reusable information needed to launch an agent: an ID, a name, spawn configuration, and a runtime provider.

## Template structure

```typescript
{
  id: "example",
  name: "Example agent",
  spawn: {
    command: "pnpm",
    args: ["exec", "tsx", "./agent.ts"],
  },
  runtime: { provider: "local" },
}
```

## Built-in templates

Flamecast ships with two built-in templates:

| Template | Runtime | Description |
|---|---|---|
| `example` | `local` | Spawns an agent as a local child process |
| `example-docker` | `docker` | Builds and runs an agent inside a Docker container |

## Register custom templates

### At construction time

Pass templates to the `Flamecast` constructor. This replaces the built-in defaults:

```typescript
const flamecast = new Flamecast({
  agentTemplates: [
    {
      id: "my-agent",
      name: "My custom agent",
      spawn: { command: "node", args: ["agent.js"] },
      runtime: { provider: "local" },
    },
  ],
});
```

### Via the API

`POST /api/agent-templates` registers additional templates in storage, so they survive restarts as long as the storage backend is durable.

```bash
curl -X POST http://localhost:3001/api/agent-templates \
  -H "Content-Type: application/json" \
  -d '{
    "id": "my-agent",
    "name": "My custom agent",
    "spawn": { "command": "node", "args": ["agent.js"] },
    "runtime": { "provider": "local" }
  }'
```

## Creating sessions from templates

Use the `agentTemplateId` field when creating a session:

```bash
curl -X POST http://localhost:3001/api/agents \
  -H "Content-Type: application/json" \
  -d '{ "agentTemplateId": "my-agent" }'
```

You can also create a one-off session without a template:

```bash
curl -X POST http://localhost:3001/api/agents \
  -H "Content-Type: application/json" \
  -d '{
    "spawn": { "command": "node", "args": ["agent.js"] },
    "name": "Scratch agent"
  }'
```
