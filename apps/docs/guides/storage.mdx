Flamecast persists session metadata and agent templates through SQL-backed storage adapters.

## Backends

| Backend | Package | Use case |
|---|---|---|
| PGLite | `@flamecast/storage-psql` | Embedded Postgres. No external database needed |
| PostgreSQL | `@flamecast/storage-psql` | Production. Full Postgres with connection pooling |

## Configuration

`Flamecast` can be configured with either a ready `storage` instance or a lazy SQL `backend` handle. In Node environments, the preferred pattern is exporting a configured `Flamecast` instance backed by `createPsqlDatabase()`.

If you want `flamecast db migrate` to reuse the same database settings as your app, put them in `flamecast.config.ts` and import that module from your server entrypoint:

```typescript
import dotenv from "dotenv";
import { Flamecast, NodeRuntime } from "@flamecast/sdk";
import { createPsqlDatabase } from "@flamecast/storage-psql";

dotenv.config();
const db = createPsqlDatabase({
  url: process.env.DATABASE_URL,
});

export default new Flamecast({
  backend: db,
  runtimes: {
    default: new NodeRuntime(),
  },
});
```

### PGLite (default Node path)

```typescript
import { Flamecast, NodeRuntime } from "@flamecast/sdk";
import { createPsqlStorage } from "@flamecast/storage-psql";

const flamecast = new Flamecast({
  storage: await createPsqlStorage(),
  runtimes: {
    default: new NodeRuntime(),
  },
});
```

PGLite stores data in `<cwd>/.flamecast/pglite` by default. Override with the `FLAMECAST_PGLITE_DIR` environment variable or the `dataDir` option.

Before first startup, apply migrations explicitly:

```bash
flamecast db migrate
```

The CLI auto-loads `flamecast.config.ts|mts|js|mjs|cts|cjs` from the current working directory. The preferred pattern is to export a configured `Flamecast` instance backed by a SQL adapter. Use `--config <path>` to point somewhere else.

### PostgreSQL

```typescript
import { Flamecast, NodeRuntime } from "@flamecast/sdk";
import { createPsqlDatabase } from "@flamecast/storage-psql";

const flamecast = new Flamecast({
  backend: createPsqlDatabase({
    url: "postgres://localhost/flamecast",
  }),
  runtimes: {
    default: new NodeRuntime(),
  },
});
```

## Options

| Option | Description |
|---|---|
| `url` | Postgres connection string |
| `dataDir` | PGLite data directory (default: `<cwd>/.flamecast/pglite`) |

The current SQL storage package uses Drizzle ORM under the hood. Check or apply runtime migrations with:

```bash
flamecast db status
flamecast db migrate
flamecast db studio
```

Generate new migration files during development with:

```bash
pnpm psql:generate
```
