The `@flamecast/sdk/client` package provides a typed client for interacting with Flamecast from the browser or Node.js.

## REST client

```typescript
import { createFlamecastClient } from "@flamecast/sdk/client";

const client = createFlamecastClient({ baseUrl: "http://localhost:3001" });

// List templates
const templates = await client.fetchAgentTemplates();

// Create a session
const session = await client.createSession({ agentTemplateId: "example" });

// Get session details
const snapshot = await client.fetchSession(session.id);

// Terminate
await client.terminateSession(session.id);
```

## FlamecastSession

`FlamecastSession` is an auto-reconnecting WebSocket wrapper for a single session.

<Note>
  ***Import path:*** `FlamecastSession` is not yet re-exported from `@flamecast/sdk/client`. Import it directly from the internal module path for now:
</Note>

```typescript
import { FlamecastSession } from "@flamecast/sdk/client/lib/flamecast-session";

const session = new FlamecastSession({
  websocketUrl: "ws://localhost:3001/ws/sessions/abc-123",
  sessionId: "abc-123",
});

session.connect();
```

### Listen for events

```typescript
const unsubscribe = session.on((event) => {
  console.log("Event:", event.type, event.data);
});
```

### Send commands

```typescript
// Send a prompt
session.prompt("Hello, agent!");

// Respond to a permission request
session.respondToPermission(requestId, { optionId: "allow" });

// Cancel a queued prompt
session.cancel(queueId);

// Terminate the session
session.terminate();

// Disconnect the WebSocket
session.disconnect();
```

### Read files

```typescript
const { content, truncated } = await session.requestFilePreview("/src/index.ts");
```

### Connection state

The `connectionState` property reflects the current WebSocket status:

| State | Description |
|---|---|
| `disconnected` | Not connected |
| `connecting` | Initial connection in progress |
| `connected` | Active connection |
| `reconnecting` | Attempting to reconnect after a drop |

Listen for state changes:

```typescript
session.onStateChange((state) => {
  console.log("Connection:", state);
});
```

### Options

| Option | Default | Description |
|---|---|---|
| `websocketUrl` | Required | WebSocket URL for the session |
| `sessionId` | Required | Session identifier |
| `maxReconnectAttempts` | `5` | Number of reconnection attempts before giving up |

## React hooks

The SDK includes a React hook for use with TanStack Query:

<Note>
  ***Import path:*** `useFlamecastSession` is not yet re-exported from `@flamecast/sdk/client`. Import it directly for now:
</Note>

```typescript
import { useFlamecastSession } from "@flamecast/sdk/client/hooks/use-flamecast-session";

function SessionView({ sessionId }: { sessionId: string }) {
  const {
    events,
    connectionState,
    isConnected,
    prompt,
    respondToPermission,
    cancel,
    terminate,
  } = useFlamecastSession(sessionId);

  return (
    <div>
      <p>Status: {connectionState}</p>
      <p>Events: {events.length}</p>
      <button onClick={() => prompt("Hello!")}>Send prompt</button>
    </div>
  );
}
```

The hook manages connection lifecycle, event buffering, and state synchronization automatically.
